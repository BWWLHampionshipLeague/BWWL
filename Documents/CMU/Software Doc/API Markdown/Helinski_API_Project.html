GetMyWeather API

This document will provide reference for using the GetMyWeather API to embed a weather forecast onto a web page.

GetMyWeather forecasts are unique in that they are considered a "micro-forecast". Learn how to customize your results to provide highly specific updates to your web page.

Overview

Each web session requires you to first initialize GetMyWeather. You then use the getWeather method to retrieve each forecast. The methods returns an HTML snippet, which can be processed with Javascript and/or styled with CSS.

GetMyWeather is supported by all modern browsers that support Javascript, which must be enabled to use GetMyWeather. The return package is static, and you must invoke getWeather again with new parameter values for an update.

Prerequisites

Obtain Your API Key

  1. Register for an API key at https://getmyweather.com Cost depends on usage; with details available on the website. A no-cost key is available for application development and testing.

Insert HTML Return Destination

  1. Copy the following empty element into your HTML page. The getWeather return package will be inserted here.

    <div id="forecast"></div>

Parameters

GetMyWeather uses four parameters to produce a result. They are

  • Latitude

  • Longitude

  • Specificity

  • Time

The following table explains the parameters and their legal values.

Path Parameters

NameData TypeMin - MaxRequiredDescription
LatitudeDoubleYesLocation based on latitude
LongitudeDoubleYesLocation based on longitude
SpecificityInteger10 - 100,000Defines the area of the forecast as a circle around the location
TimeIntegerTime in the future for which the forecast is requested. Expressed in hours, decimals are allowed

Request Format and Results

To call the GetMyWeather method:

  1. Input your parameters into the following code

 <script async defer
    src="https://www.getmyweather.com/getWeather?
    key=<YOUR_KEY>&
    callback=init&
    location=<LATITUDE>:<LONGITUDE>&
    specificity=<SPECIFICITY>&
    time=<TIME>">
    </script>

Upon success, the getWeather function returns the following HTML structure (values are samples).

 <div class="forecast">
    <div class="temperature">78</div>
    <div class="windspeed">15</div>
    <div class="chanceRain">30</div>
    <div class="trust">80</div>
    </div>

Return Parameters

The following table explains each of the return parameters.

NameData TypeMin - MaxDescription
temperatureIntegerTemperature in Fahrenheit
windspeedIntegerWind Speed in Miles per Hour
chanceRainInteger0 - 100Chance of rain expressed as a percentage
TrustInteger1 - 100Represents the reliability of the forecast (1 is very unreliable; 100 is absolutely reliable

Error Conditions

The following error conditions may arise.

  • Invalid Key API Key invalid. Check GetMyWeather.com for details

  • Invalid parameterformat Parameter input is the wrong data type

  • Parameter out of range Parameter input is out of min - max range

  • Server unavailable If no response, check GetMyWeather.com for updates
